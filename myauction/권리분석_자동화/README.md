# 권리분석 보증서·브리핑 자동화 리소스

초보자과정 교재(`경매교재-초보자과정_정리.md`)에서 파생한 **자동화용 데이터·템플릿 묶음**입니다.
교재(사람이 읽는 지식)를 3개 레이어로 분리해, 자동화가 "정확하게" 동작하도록 설계했습니다.

## 왜 이렇게 나눴나

권리분석 보증서는 틀리면 안 되는 문서입니다. 말소기준·배당순위·소액임차인 금액을 LLM 문장 추론에 맡기면 환각 위험이 있으므로,
**판단 로직·기준표는 코드가 참조하는 구조화 데이터**로, **설명·해석은 텍스트 참조**로 분리했습니다.

```
[입력]  등기부등본 + 매각물건명세서 + 현황조사서
          │  (크롤링/파싱 → 입력_스키마.json 형태로 정규화)
          ▼
[규칙엔진]  권리분석_규칙.json  +  기준표_소액임차인.json
          │  · 말소기준권리 확정 → 인수/소멸 판정
          │  · 임차인 4형태 판정 → 대항력/우선변제/최우선변제 대상 O/X
          │  · 배당순위로 예상 배당표 계산(최우선변제 실제 금액은 별도 검토)
          ▼
[산출]  ├─ 보증서_템플릿.md   (사실·판단 중심 + 근거)
        └─ 브리핑_템플릿.md   (쉬운 설명 중심, 용어사전 활용)
                    ▲
        권리분석_용어사전.json  (해석·브리핑 문구 생성 근거)
        경매교재-초보자과정_정리.md  (RAG 참조 지식베이스)
```

## 파일 구성

| 파일 | 레이어 | 용도 |
| --- | --- | --- |
| `권리분석_규칙.json` | ① 결정 로직 | 말소기준·인수소멸·대항력·우선변제·최우선변제·배당순위·상가. **보증서 정확성의 핵심** |
| `기준표_소액임차인.json` | ② 룩업 데이터 | 시점·지역별 소액임차인 보증금상한 조회 및 최우선변제 대상 O/X/- 판정 |
| `권리분석_용어사전.json` | ③ 텍스트 | 용어-풀이 28개. 브리핑·설명 문구, 검색 |
| `입력_스키마.json` | 계약 | 파이프라인 입력 형태 정의(파서 ↔ 규칙엔진) |
| `보증서_템플릿.md` | 산출 | 변수 치환용 보증서 골격({{ }} 자리표시자) |
| `브리핑_템플릿.md` | 산출 | 고객용 요약 브리핑 골격 |
| `경매교재-초보자과정_정리.md` | 지식베이스 | 전체 교재. RAG 청킹 원본 |

## 데이터 흐름 상세

1. **파싱**: 경매사이트/등기부 PDF → `입력_스키마.json` 구조로 정규화. `권리코드`(KEUN_MORTGAGE 등)를 부여.
2. **말소기준 확정**: 을구+갑구에서 `말소기준권리.후보권리` 코드에 해당하는 것 중 접수일 최선순위 1개 선택.
3. **인수/소멸**: 각 권리 성립일을 말소기준 설정일과 비교(`인수소멸_판별`). 예고등기·유치권 등은 별도 경고.
4. **임차인 판정**: 전입/확정일자/보증금 → 대항력 발생일(전입 다음날 0시) 계산 → 말소기준 대비 선후 → `임차인_4형태` 매트릭스.
5. **최우선변제**: 최초 담보물권 설정일 = 기준일 → `기준표_소액임차인.json`에서 지역·시점 구간 조회 → 소액임차인 대상 가능성 O/X/-. 실제 변제금액·주택가액 1/2 한도·배당가능액은 별도 배당 계산에서 검토.
6. **배당표**: `배당순위`로 순위 배당(안분/흡수) 계산.
7. **문서화**: 산출값을 `보증서_템플릿.md`/`브리핑_템플릿.md`의 {{변수}}에 치환. 브리핑 설명문은 용어사전·교재를 근거로 생성.

## ⚠️ 반드시 지킬 것

- **금액 검증**: `기준표_소액임차인.json`의 값은 국가법령정보센터(law.go.kr) 시행령 원문으로 대조 후 확정. 미검증 구간은 `검증필요: true`.
- **사람 검토 단계**: 보증서는 자동 초안 → 담당자 검토·서명 후 발급. 템플릿에 검토란·면책문구 포함.
- **미공시 권리**: 유치권·법정지상권 등은 등기부에 없으므로 자동 판정 한계. 현황조사서·경고 항목으로 보완.
- **원문 보존**: 규칙 문구는 교재 취지를 보존. 임의 변경 금지.

## RAG 청킹 권장

`경매교재-초보자과정_정리.md`를 벡터DB에 넣을 때는 **`###`(소절) 단위 청킹**을 권장합니다.
각 청크에 메타데이터로 `강`, `절`, `소절`, 별점(★) 중요도를 부착하면 검색 정확도가 올라갑니다.
용어 질의는 `권리분석_용어사전.json`을 우선 조회하고, 서술형 해석은 교재 청크를 검색하는 2단 구조가 효율적입니다.
